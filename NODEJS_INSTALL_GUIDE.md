# Инструкция по установке Node.js 7.2.3

## ⚠️ ВАЖНОЕ ПРЕДУПРЕЖДЕНИЕ

Node.js 7.2.3 — это **очень старая версия** (2016 год), которая:
- ❌ **Не поддерживается** (End of Life с 2017 года)
- ❌ **Содержит критические уязвимости безопасности**
- ❌ **Не работает с современными npm пакетами**
- ❌ **Может вызвать проблемы совместимости**

**Текущая версия:** v24.11.1 ✅

**Рекомендация:** Используйте текущую версию или LTS (v20.x / v22.x)

---

## Вариант 1: Использование nvm-windows (рекомендуется)

### Шаг 1: Установка nvm-windows

1. Скачайте установщик с GitHub: https://github.com/coreybutler/nvm-windows/releases
2. Запустите `nvm-setup.exe`
3. Перезапустите PowerShell/CMD

### Шаг 2: Установка Node.js 7.2.3

```powershell
# Установка Node.js 7.2.3
nvm install 7.2.3

# Переключение на версию 7.2.3
nvm use 7.2.3

# Проверка версии
node --version
```

### Шаг 3: Возврат к текущей версии

```powershell
# Переключение обратно на v24.11.1
nvm use 24.11.1
```

---

## Вариант 2: Прямая установка (не рекомендуется)

1. Скачайте архивную версию с официального сайта:
   - https://nodejs.org/dist/v7.2.3/
   - Выберите `node-v7.2.3-x64.msi` для Windows 64-bit

2. ⚠️ **ВНИМАНИЕ:** Это перезапишет текущую версию Node.js!

3. После установки проверьте:
   ```powershell
   node --version
   ```

---

## Вариант 3: Использование Docker (безопасно)

Если вам нужна именно эта версия для тестирования, используйте Docker:

```powershell
# Запуск контейнера с Node.js 7.2.3
docker run -it --rm -v ${PWD}:/app -w /app node:7.2.3 bash

# Внутри контейнера
node --version
npm install
```

---

## Почему не стоит использовать Node.js 7.2.3?

1. **Безопасность:** Известные уязвимости (CVE), которые не будут исправлены
2. **Совместимость:** Современные пакеты требуют Node.js 14+ или 16+
3. **Производительность:** Старые версии работают медленнее
4. **Поддержка:** Нет обновлений и исправлений ошибок

---

## Альтернативное решение

Если вам нужна старая версия для совместимости со старым кодом:

1. Используйте **nvm-windows** для переключения между версиями
2. Или используйте **Docker** для изоляции старой версии
3. Или **обновите код** для работы с современными версиями Node.js
